Leetcode 1: Two Sum

The Two Sum Problem üéØ

Two Sum is a classical problem where you‚Äôre given a number array and a target, and you need to find two indices from the array where the numbers add up to the target. üî¢

Let’s try to solve it üß©

  • You can check all possible pairs, which leads to an algorithm with quadratic complexity, or you can use a hash table to bring the complexity down to linear. üìâ

    For example you have nums = [2, 5, 7, 8] and target = 9, listing all possibilities will be like

    [[2, 2][2,5][2,7][2,8]]
[[5, 2][5,5][5,7][5,8]]
[[7, 2][7,5][7,7][7,8]]
[[8, 2][8,5][8,7][8,8]]
It like 4x4 i.e. n x n -> O(n^2)

  • Let‚Äôs go with the second method, where you achieve linear complexity by using a hash table. üóÉÔ∏è
    A hash table is a data structure that allows inserting and retrieving key-value pairs in constant time. ‚ö°

  • Iterate over the number list and, for each number, record its complement and index if no number in the table can act as a complement for the current number.

  • If a complement for the current number exists, retrieve its index and return it along with the current index as the answer. ‚úÖ

  • For the edge case where no pair is found, return a -1, -1 pair. ‚ùå

That‚Äôs it! Now, let‚Äôs break down the code. üßë‚Äçüíª

Code:

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        map<int, int> mp;
        for(int i = 0; i < nums.size(); i++) {
            if(mp.count(nums[i])) {
                return { mp[nums[i]], i };
            } else {
                mp[target - nums[i]] = i;
            }
        }
        return {-1, -1};
    }
};

Understanding Time and Space Complexity ‚è±Ô∏èüìè

Time Complexity:

We iterate over each element in the array once, performing constant-time operations (insertion and retrieval) on each iteration. This results in a linear O(n) time complexity. üöÄ

Space Complexity:

We use a hash map to store up to n pairs of elements, so the space complexity is also linear, O(n), which is a trade-off for faster lookup. üóÑÔ∏è
