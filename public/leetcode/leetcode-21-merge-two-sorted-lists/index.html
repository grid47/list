Leetcode 21: Merge Two Sorted Lists

Code:

/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
        if(l1 == NULL) return l2;
        if(l2 == NULL) return l1;
        
        if(l1->val < l2->val) {
            l1->next = mergeTwoLists(l1->next, l2);
            return l1;
        }
        l2->next = mergeTwoLists(l1, l2->next);
        return l2;            
    }
};

üìù The Problem

We are given two sorted singly linked lists, l1 and l2, and our goal is to merge them into a single, sorted linked list. Both lists are already in non-decreasing order, so we just need to combine them while preserving that order.

üîë Key Idea: Think Recursively!

We‚Äôll approach this problem using recursion‚Äîa technique that breaks a complex problem into simpler subproblems. The idea is to always pick the smaller value between the two lists and recursively merge the remaining elements. By doing this, we ensure that the merged list remains sorted.

  1. Base Case: If one of the lists is NULL, simply return the other list. This is because if one list is empty, there‚Äôs nothing to merge, so we just return the other sorted list.

if(l2 == NULL) return l1;

  2. Recursive Case: Now, we compare the values of the current nodes in l1 and l2. We then take the smaller node and recursively merge the rest of the lists.

    if(l1->val < l2->val) {
    l1->next = mergeTwoLists(l1->next, l2);
    return l1;
}
    • Explanation Simply: If the current node in l1 has a smaller value, we attach it to the merged list and continue merging the remaining l1 with l2.

  3. If the Current Node of l2 is Smaller: If l2 has a smaller or equal value, we do the same thing for l2:

    l2->next = mergeTwoLists(l1, l2->next);
return l2;

üß† Time and Space Complexity

  • Time Complexity:
    O(n + m) ‚Äî where n and m are the lengths of l1 and l2, respectively. We traverse both lists once, making this solution linear.

  • Space Complexity:
    O(n + m) ‚Äî due to the recursion call stack. In the worst case, the maximum depth of the recursion is the total number of nodes in both lists.

üí° Why Recursion Rocks!

  • Optimal Performance: We make sure to visit each node once and do a simple comparison at each step, making this solution both efficient and easy to understand.
