Leetcode 2100: Find Good Days to Rob the Bank

grid47
Exploring patterns and algorithms
Apr 11, 2024 7 min read

You are planning a bank robbery with a gang of thieves. You are given a list, ‘security’, where each element represents the number of guards present on duty each day. You are also given an integer, ’time’, which represents the number of days before and after the current day that must have specific guard arrangements for it to be a good day for the robbery. A good day to rob the bank is defined as:

  1. There must be at least ’time’ days before and after the current day.
  2. The number of guards on the days before the current day must be non-increasing.
  3. The number of guards on the days after the current day must be non-decreasing. Return a list of all the days that are good days to rob the bank.

Approach: The solution involves computing two auxiliary arrays: one for the number of consecutive non-increasing days before each day and another for the number of consecutive non-decreasing days after each day.
Observations:
• We need to ensure that we correctly identify the days with appropriate guard patterns before and after them.
• We will use two auxiliary arrays to track the number of non-increasing days before and non-decreasing days after each day, which will allow us to efficiently check each day.
Steps:
• Initialize two arrays: 'pre' for the number of non-increasing days before each day and 'suf' for the number of non-decreasing days after each day.
• Fill these arrays by iterating through the 'security' array from left to right for 'pre' and right to left for 'suf'.
• Iterate through the 'security' array and for each day, check if it satisfies the conditions for being a good day to rob the bank using the 'pre' and 'suf' arrays.

• There are no edge cases for empty inputs as the problem guarantees that the input array will not be empty.
Large Inputs:
• The solution needs to handle arrays of up to 10^5 elements efficiently.
Special Values:
• Ensure the solution handles cases where 'time' is 0, which means every day is a good day.
Constraints:
• The solution must operate within the time complexity of O(n), where n is the length of the array.
vector<int> goodDaysToRobBank(vector<int>& sec, int time) {

    int n = sec.size();
    vector<int> pre(n, 0), suf(n, 0);
    pre[0] = 0;
    int cnt = 0;
    for(int i = 1; i < n; i++) {
        if(sec[i] <= sec[i - 1])
            cnt++;
        else cnt = 0;
        pre[i] = cnt;
    }
    
    suf[n - 1] = 0;
    cnt = 0;
    for (int i = n - 2; i >= 0; i--) {
        if(sec[i] <= sec[i + 1])
            cnt++;
        else cnt = 0;
        suf[i] = cnt;
    }
    
    vector<int> ans;
    for (int i = 0; i < n; i++) {
        if (pre[i] >= time && suf[i] >= time)
            ans.push_back(i);
    }
    
    return ans;
}
