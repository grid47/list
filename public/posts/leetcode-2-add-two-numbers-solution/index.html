Code:

class Solution {
public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
        ListNode* head = new ListNode(0);
        ListNode* tail = head;
        int rm = 0;

        while (l1 != NULL || l2 != NULL || rm != 0) {
            int no1 = (l1 != NULL) ? l1->val : 0;
            int no2 = (l2 != NULL) ? l2->val : 0;

            int sum = no1 + no2 + rm;
            int digit = sum % 10;
            rm = sum / 10;

            ListNode* nxt = new ListNode(digit);
            tail->next = nxt;
            tail = tail->next;

            l1 = (l1 != NULL) ? l1->next : NULL;
            l2 = (l2 != NULL) ? l2->next : NULL;
        }

        ListNode* res = head->next;
        delete head;
        return res;
    }
};

The Add Two Numbers problem is a classic linked list challenge that is both intuitive and powerful. In this problem, two numbers are represented by linked lists in reverse order, and the task is to add these numbers and return the sum as a linked list in the same reverse order. Let‚Äôs dive into how to tackle this step by step!

üìù Problem Statement Link to heading

You are given:

  • Two linked lists, l1 and l2, where each node contains a single digit of a non-negative integer, stored in reverse order. This means the least significant digit is at the head of the list.
  • The goal is to add these two numbers and return the result as a linked list, also in reverse order.

üîç Plan and Approach: Adding Digit by Digit with Carry Link to heading

We‚Äôll use a method that mirrors manual addition:

  1. Traverse both linked lists: We‚Äôll add corresponding digits from l1 and l2, handling any remaining digits if the lists are of different lengths.
  2. Carry management: Keep track of any carry-over from each addition, carrying it to the next higher digit.
  3. Build the result list: As we sum digits, we‚Äôll build the result list node-by-node in reverse order, just like the input format.

Step-by-Step Solution Link to heading

Step 1: Initialize Variables Link to heading 

ListNode* head = new ListNode(0);  // Dummy node to start the result list
ListNode* tail = head;
int carry = 0;
  • head acts as a dummy node to simplify list operations.
  • tail will point to the last node in our result list, making it easy to append new nodes.
  • carry will store any overflow value from each digit addition.

Step 2: Traverse and Add Corresponding Digits Link to heading 

while (l1 != NULL || l2 != NULL || carry != 0) {
    int no1 = (l1 != NULL) ? l1->val : 0;
    int no2 = (l2 != NULL) ? l2->val : 0;

    int sum = no1 + no2 + carry;
    int digit = sum % 10;
    carry = sum / 10;

    ListNode* nxt = new ListNode(digit);
    tail->next = nxt;
    tail = tail->next;

    l1 = (l1 != NULL) ? l1->next : NULL;
    l2 = (l2 != NULL) ? l2->next : NULL;
}
  • We loop while there are remaining digits in l1 or l2, or there‚Äôs a carry left.
  • For each iteration:
    • Get the value of each list‚Äôs current node (0 if the list has ended).
    • Compute sum as the total of these values plus the carry.
    • Extract the resulting digit (sum modulo 10) and update carry (sum divided by 10).
    • Append digit to our result list and advance pointers.

Step 3: Return the Result List Link to heading 

ListNode* res = head->next;
delete head;
return res;
  • After the loop, head->next points to the result list, skipping the dummy node.
  • We delete the dummy node to prevent memory leaks.
  • The resulting list represents the sum, stored in reverse order.

üåü Handling Edge Cases Link to heading

  • Different Lengths: If one list is shorter, its remaining nodes are considered 0.
  • Final Carry: If there‚Äôs still a carry after processing both lists, a new node with this carry value is appended.

‚è±Ô∏è Understanding Time and Space Complexity Link to heading

  • Time Complexity: O(max(m, n)), where m and n are the lengths of l1 and l2. We iterate through each node once, resulting in linear time complexity.
  • Space Complexity: O(max(m, n)) for storing the result list, with an additional node if there‚Äôs a carry at the end.

üí° Why This Approach Works Well Link to heading

This solution mimics manual addition closely and is both efficient and intuitive. The dummy node simplifies list construction, avoiding special cases for the head of the list. By following a structured approach, we maintain clean and readable code.
